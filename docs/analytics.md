# 洞察分析页面

## 概述

洞察分析页面将用户的求职数据转化为可视化的洞察报告，帮助用户从"被动记录"升级为"主动优化"。通过漏斗分析、趋势追踪、来源分析等维度，让用户看清自己的求职效率和瓶颈，从而调整策略。这是本产品的差异化功能，解决"投了没回音"焦虑的最佳方式不是让用户投更多，而是让用户投得更聪明。

## 页面头部

- **标题**："📈 求职洞察"（H1 24px/600）
- **时间范围选择器**：右上角下拉选择
  - 选项：本周 / 本月 / 本季度 / 全部 / 自定义范围
  - 默认：本月
  - 样式：下拉选择框，height 36px，圆角 sm，背景 `#FFFFFF`，边框 1px `#E8EBF0`
- **数据更新时间**：Caption/Text Tertiary，"数据更新于 3月12日 15:30"

## 核心指标卡片区

页面顶部横向排列 4 个核心指标卡片，等宽分布，间距 16px。

### 指标卡片样式

- **尺寸**：flex 1，height 100px，padding 16px，圆角 lg，背景 `#FFFFFF`，阴影 shadow-sm
- **结构**：
  - 顶部：指标名称（Caption/Text Secondary）+ 趋势指示（如 "↑ 20%" 或 "↓ 5%"，对比上个周期）
  - 中部：指标数值（Display 32px/700/Text Primary）
  - 底部：对比说明（Caption/Text Tertiary，如"较上月 +8"）
- **Hover**：阴影增强，显示"查看详情"文字（Caption/Primary Blue）

### 四个核心指标

| 指标名 | 说明 | 计算方式 | 趋势对比 |
|--------|------|----------|----------|
| **总投递** | 已投递的申请总数 | 状态 = 已投递 + 之后所有状态的计数 | 与上个周期对比 |
| **获得面试** | 进入面试流程的数量 | 状态 = 面试中 + 等结果 + 已Offer | 与上个周期对比 |
| **面试通过率** | 从投递到面试的转化率 | 获得面试 ÷ 总投递 × 100% | 与上个周期对比 |
| **待处理** | 当前需要关注的申请 | 状态 = 准备中 + 已投递 + 笔试中 + 面试中 + 等结果 | 与昨天对比 |

### 指标卡片颜色编码

- 总投递：Primary Blue（#3B5BFF）数值颜色
- 获得面试：Info Cyan（#06B6D4）数值颜色
- 面试通过率：Success Green（#22C55E）数值颜色
- 待处理：Warning Orange（#F59E0B）数值颜色（如果 > 10 变为 Danger Red）

## 求职漏斗图

### 概述

以桑基图或垂直漏斗图展示求职各环节的人数流转，让用户一眼看清哪个环节是最大瓶颈。

### 布局位置

左侧卡片，占据约 55% 宽度，height 360px，圆角 lg，背景 `#FFFFFF`，阴影 shadow-sm，padding 20px。

### 图表设计

**漏斗图（垂直堆叠）**：
- 从上到下 7 个层级，对应 7 种状态
- 每个层级一个横向色条，宽度 = 该状态任务数 / 最大状态任务数 × 100%
- 色条使用对应状态色（准备中灰/已投递蓝/笔试中橙/面试中青/等结果紫/已Offer绿/已拒绝红）
- 色条右侧显示：状态名（14px/600）+ 数量（14px/Text Secondary）+ 转化率（Caption/Text Tertiary，如"28% 进入下一环节"）
- 色条之间有向下箭头连接（↓，24px/Text Tertiary）
- 最宽的一条（通常是"已投递"或"准备中"）为 100% 宽度，其余按比例缩放

**交互**：
- Hover 某层级：该层级色条宽度轻微放大（scaleX 1.02），显示 tooltip 包含详细数据（数量、转化率、平均停留天数）
- 点击某层级：跳转到看板总览的对应状态列

### 空状态

- 无数据时显示："还没有足够的数据生成漏斗图" + "开始记录求职任务后，这里会展示你的求职管道分析"

## 每周投递趋势

### 概述

折线图展示每周的投递数量和面试数量变化趋势，帮助用户了解自己的求职节奏是否合理。

### 布局位置

右侧卡片，占据约 43% 宽度（与漏斗图同行），height 360px，相同样式。

### 图表设计

**双折线图**：
- X轴：周次（如"第1周"、"第2周"... 或具体日期范围）
- Y轴：数量（0 到最大值+5）
- **投递线**：Primary Blue（#3B5BFF），2px 实线，圆点数据标记（6px）
- **面试线**：Success Green（#22C55E），2px 实线，圆点数据标记（6px）
- **面积填充**：折线下方填充对应颜色的 10% 透明度
- **网格线**：横向虚线（1px `#F1F3F5`）
- **当前周期指示**：最后一周的数据点有脉动动画（2s infinite），表示"进行中"

**交互**：
- Hover 数据点：tooltip 显示该周具体日期范围 + 投递数 + 面试数
- 点击数据点：跳转到对应周的周历视图

### 周次计算

- 以求职季开始日期（用户首次创建任务日期）或自定义起始日期为第1周
- 每7天为一周
- 当前周标记为"本周"（如果还在求职季内）

## 状态分布环形图

### 概述

环形图（甜甜圈图）展示当前所有任务在各状态中的分布比例，让用户了解自己的求职组合是否健康。

### 布局位置

左下卡片，占据约 40% 宽度，height 280px。

### 图表设计

- **中心区域**：总任务数（Display 28px/700）+ "总任务"（Caption）
- **环形**：按状态比例分割，每段使用对应状态色
- **环形宽度**：24px
- **图例**：环形右侧，每行一个状态色圆点（8px）+ 状态名 + 数量 + 百分比
- **空状态**：只有一个灰色环 + "还没有任务"文字

**交互**：
- Hover 某段：该段外扩（半径+4px），其他段 opacity 0.5，tooltip 显示数量和百分比
- 点击某段：跳转到搜索页面，自动按该状态筛选

## 标签分析柱状图

### 概述

横向柱状图展示各标签下的任务数量和通过率，帮助用户了解哪类公司/岗位投递效果最佳。

### 布局位置

右下卡片，占据约 58% 宽度（与环形图同行），height 280px。

### 图表设计

- **横向柱状图**：每个标签一行
- **左轴**：标签名称（带彩色圆点），最多显示前8个标签
- **柱状**：
  - 蓝色段：已投递数量
  - 绿色段：已获得面试/Offer的数量（叠加在蓝色段上，显示转化率）
- **X轴**：数量
- **右侧数据**：转化率百分比（如"45%"，只显示 > 0% 的）
- **排序**：默认按总数量降序，可切换为按转化率降序

**交互**：
- Hover 某行：该行背景 Surface Hover，tooltip 显示详细数据
- 点击某行：跳转到搜索页面，按该标签筛选

## 即将截止任务提醒

### 概述

页面底部的横向列表，展示截止日期临近的任务，以时间线形式排列，这是从分析回到行动的关键桥梁。

### 布局位置

页面底部全宽卡片，height 自适应（最小 120px），padding 20px。

### 内容设计

- **标题**："⏰ 时间紧迫"（H3 15px/600）+ 右侧"查看全部"链接（跳转搜索"即将截止"筛选）
- **时间线**：横向滚动
  - 今日（红色标记线）→ 明天 → 后天 → ... 未来7天
  - 每个日期下方：该日期有截止任务的公司名缩略卡片（公司名首字母+状态色背景，32px圆形）
  - 如果某天有多个任务，显示"+N"重叠标记
- **任务卡片**（Hover 日期时展开）：
  - 公司名称 + 岗位名称
  - 截止时间（精确到小时）
  - 状态徽章
  - "查看详情"链接

### 紧急度视觉编码

- < 24小时：红色脉动边框
- 24-72小时：橙色边框
- 3-7天：黄色边框
- > 7天：无边框，正常显示

## 备注

### 数据计算逻辑

**漏斗转化率计算**：
- 准备中 → 已投递：在准备中停留后变为已投递的比例
- 已投递 → 笔试中：已投递后收到笔试的比例
- 笔试中 → 面试中：笔试通过后进入面试的比例
- 面试中 → 等结果：面试完成后等待结果的比例
- 等结果 → 已Offer：最终收到Offer的比例
- 已拒绝 = 任意状态变为已拒绝的比例（分流计算）

**周次定义**：
- 以用户设置的起始日期为基准（默认为首次创建任务的日期）
- 用户可在设置中修改"求职季起始日期"
- 自然周（周一到周日）

### 智能建议（扩展）

基于数据分析，系统可以在页面底部提供智能建议卡片：

- 如果面试通过率 < 10%："你的面试转化率偏低，建议优化简历或尝试内推渠道"
- 如果"等结果"状态堆积 > 5 个："有多个申请在等待反馈，建议主动发邮件跟进"
- 如果某标签通过率明显高于其他："互联网大厂类申请的通过率最高，可以适当增加此类投递"
- 如果本周投递量为0："本周还没有新的投递，保持节奏很重要"

建议卡片样式：
- 背景：Info Cyan 5%（信息类）或 Warning Orange 5%（提醒类）
- 左侧：对应颜色的竖条（3px）
- 内容：建议文字（14px/Text Primary）+ "了解更多"链接
- 可关闭（×按钮），关闭后24小时内不再显示同类建议

### 数据导出

页面右上角提供"导出报告"按钮：
- 点击后弹出选项：导出为 PDF / 导出为图片 / 导出为 CSV 数据
- PDF：生成格式化的求职报告，适合打印或发送给职业顾问
- 图片：生成所有图表的合成图片，适合分享到社交媒体（去敏处理后）
- CSV：原始数据，适合进一步分析

### 与其他页面的联动

洞察分析不是孤立页面，而是与整个产品形成闭环：
- 漏斗图 → 点击跳转到看板总览
- 折线图 → 点击跳转到周历视图
- 环形图/柱状图 → 点击跳转到搜索筛选
- 即将截止 → 点击打开任务详情
- 每个联动都在新标签页/视图打开，保留洞察页面的状态

### 空数据策略

当用户刚注册、任务数量 < 5 时：
- 显示"示例数据"模式：用模拟数据填充图表，让用户理解洞察页面的价值
- 图表上方半透明遮罩 + "这是示例数据，添加5个以上任务后查看你的真实分析"文字
- 示例数据用较浅的颜色（50%透明度），与真实数据区分
