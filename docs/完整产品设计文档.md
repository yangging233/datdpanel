# 求职申请管理看板 — 产品设计文档

## 项目信息

- **项目名称**：求职申请管理看板
- **产品定位**：面向大学生和求职者的个人求职全流程管理工具，以日历时间视图为核心，整合任务管理、进度追踪、智能提醒与数据复盘
- **目标用户**：大学生（应届生/在校生）、职场新人、正在求职的职场人士
- **使用场景**：求职季投递管理、面试进度追踪、材料截止时间监控、求职策略复盘优化

## 设计目标

1. **让时间线成为求职的指挥官**：以四种时间视图（月/周/日/时间轴）为核心载体，将散落在邮件、备忘录、网盘中的求职信息统一到一条清晰的时间线上
2. **消除"投了没回音"的焦虑黑洞**：通过可视化的进度管道和状态追踪，让用户随时掌握每个申请的真实进展
3. **从被动记录到主动驱动**：智能提醒+快捷备忘+数据分析的组合，让工具从"记录本"升级为"求职策略顾问"
4. **在大学生的认知负荷内运转**：避免专业CRM的复杂配置，用看板+日历的直觉交互降低上手门槛

## 用户痛点研究摘要

根据竞品分析和用户调研，以下痛点驱动了本次设计决策：

| 痛点 | 影响 | 设计对策 |
|------|------|----------|
| **Excel表格在15+申请后崩溃** 无法扩展，手动维护成本指数级增长 | 管理失控，漏跟机会 | 专用看板+日历视图，支持无限量申请 |
| **简历/材料版本混乱** 不同公司发送不同简历版本，无法追踪 | 发送错误版本，专业形象受损 | 每个任务独立附件管理，版本清晰 |
| **截止日期与面试提醒缺失** 依赖记忆或外部日历，容易遗漏 | 错过网申DDL、面试时间冲突 | 系统级提醒+日历可视化 deadline |
| **笔记分散在各处** 面试反馈、笔试题、复盘笔记在备忘录/微信/纸质间散落 | 需要时找不到，无法积累 | 每个任务内置笔记系统，支持标签分类 |
| **"投了没回音"的焦虑** 不知道自己的申请处于什么状态 | 心态崩溃，盲目海投 | 可视化管道+数据分析，让用户看见全貌 |
| **上下文切换成本高** 需要在邮箱、日历、云盘、笔记软件间反复跳转 | 效率低下，信息碎片化 | 一个平台整合所有求职相关信息 |

## 全局组件

### 顶部导航栏

- **高度**：56px，固定顶部（position: sticky）
- **背景**：`#FFFFFF`，底部边框 1px `#E8EBF0`
- **阴影**：滚动后出现 `shadow-sm`
- **左侧**：产品Logo（文字标识"求职看板"，16px/600/Primary Blue）+ 当前日期范围显示（如"2025年3月"，14px/Text Secondary）
- **中部**：视图切换标签组（月/周/日/时间轴），胶囊样式，选中项填充 Primary Blue + 白色文字，未选中项透明+Text Secondary
- **右侧**：
  - 搜索按钮（搜索图标，点击展开搜索栏）
  - 快捷备忘按钮（闪电图标+"速记"文字，Primary Blue边框按钮）
  - 新建任务按钮（+号图标，Primary Blue填充按钮，hover时显示下拉：普通新建/快速添加同类别/从速记导入）

### 侧边栏

- **宽度**：240px，可折叠（折叠后72px，只显示图标）
- **背景**：`#FFFFFF`，右侧边框 1px `#E8EBF0`
- **内容分区**：
  - **时间视图区**（上方，核心导航）：
    - 月历视图（日历图标+"月历"）
    - 周历视图（周图标+"周历"）
    - 日程表（列表图标+"日程表"）
    - 日时间轴（时钟图标+"日时间轴"）
    - 当前选中项：左侧 3px Primary Blue 竖线指示 + 背景 Surface Selected + 文字 Primary Blue
  - **功能区**（中部）：
    - 看板总览（看板图标+"看板总览"）
    - 洞察分析（图表图标+"洞察"）
    - 草稿箱（文件图标+"草稿箱"，右上角小红点显示未处理速记数量）
  - **标签区**（下方，可展开/折叠）：
    - 标题"任务标签"+ 展开/折叠箭头
    - 标签列表（可自定义）：互联网大厂/国企/外企/创业公司/实习/校招/内推...

### 任务卡片（日历视图内）

日历各视图中代表一个求职任务的基础单元。

- **尺寸**：根据视图自适应宽度，高度最小 28px
- **背景**：对应状态的颜色（10%透明度）
- **左侧边框**：3px 实线，对应状态颜色（100%）
- **内部**：
  - 状态色圆点（6px）+ 公司名称（12px/500/Text Primary，单行截断）
  - 岗位名称（11px/Text Secondary，单行截断）
  - 时间（11px/Text Tertiary，如"14:00"）
- **Hover**：背景变为状态色20%透明度，显示"编辑"铅笔图标（右侧）
- **点击**：打开任务详情抽屉
- **拖拽**：可拖拽到其他日期格子，拖拽中整体 opacity 0.7，缩放 1.02

### 状态徽章

- **尺寸**：height 20px，padding 0 8px，圆角 full
- **样式**：对应状态色的10%背景 + 状态色的文字
- **文字**：状态名称（如"已投递"、"面试中"）
- **变体**：小尺寸版（height 16px，padding 0 6px，用于紧凑列表）

### 优先级指示器

- **高优先级**：🔴 红色竖条（卡片左侧）+ 可选文字"高"
- **中优先级**：🟡 橙色竖条
- **低优先级**：🟢 灰色竖条（默认不显示，仅中/高显示）
- **用户自定义优先级**：用户可创建自定义优先级名称和颜色，用对应颜色的竖条表示

### 搜索组件

- **展开方式**：点击导航栏搜索按钮，从按钮位置向左展开为搜索输入框（width 240px → 400px，transition 200ms）
- **输入框**：height 36px，圆角 sm，背景 Surface，占位文字"搜索公司、岗位、标签..."
- **快捷筛选图标**：输入框右侧有筛选图标，点击展开筛选面板
- **筛选面板**（下拉）：
  - 日期范围选择器（快捷选项：今天/本周/本月/自定义）
  - 状态多选（复选框列表：准备中/已投递/笔试中/面试中/等结果/已Offer/已拒绝）
  - 标签多选（彩色圆点+标签名）
  - 优先级多选（🔴🟡🟢 + 自定义优先级）
  - 底部"清除筛选"和"应用筛选"按钮

### Toast 通知

- **位置**：右上角，距顶部 72px，距右边 16px
- **样式**：max-width 360px，padding 12px 16px，圆角 md，背景 `#FFFFFF`，阴影 shadow-lg
- **左侧**：图标（成功✅/错误❌/警告⚠️/信息ℹ️）
- **内容**：标题（14px/600）+ 描述（12px/Text Secondary）
- **动画**：从右侧滑入（translateX 100% → 0），3秒后自动淡出
- ** stacking**：最多显示3条，新的从顶部推入

## 全局交互

### 页面加载

- **骨架屏**：首次加载时显示灰色骨架矩形（模仿日历格子和卡片布局），持续 400ms 后淡入真实内容
- **视图切换**：月历↔周历↔日程表↔日时间轴 之间切换时，当前视图 opacity 1→0 淡出（150ms），新视图 opacity 0→1 淡入+轻微 translateY（8px→0，200ms）

### 新建任务流程

1. **触发方式**：
   - 点击导航栏"+"按钮
   - 在日历格子中点击空白处（自动填入该日期）
   - 右键点击日历格子（上下文菜单"新建任务"）
   - 侧边栏标签上右键"在该标签下新建"
   - 速记草稿"迁移到日历"操作
2. **新建弹窗**（居中模态框）：
   - 标题"新建求职任务"
   - 表单字段（详见 [task-detail.md](task-detail.md) 编辑表单部分）
   - 底部："保存"（Primary Blue填充）+ "保存并继续新建"（Primary Blue边框）+ "取消"
3. **快速新建（轻量版）**：
   - 在日历格子点击后出现的浮动小卡片
   - 仅包含：公司名（必填）、岗位名（必填）、日期（已填）、状态（默认"准备中"）
   - 底部"更多选项"可展开完整表单
   - 点击外部自动保存（仅必填项时）

### 拖拽与移动

- **日期调整**：在月历/周历视图中，拖拽任务卡片到另一个日期格子，释放后更新任务开始日期
- **状态变更**：在看板总览视图中，拖拽卡片到不同状态列，释放后更新状态
- **反馈**：拖拽开始时卡片 opacity 0.7 + scale 1.02 + 阴影增强；经过可放置区域时该区域显示虚线边框（2px dashed Primary Blue，背景 Primary Light）；释放后 Toast 提示"已移动到 [新日期/新状态]"

### 默认视图设置

- **设置入口**：点击用户头像 → "默认视图偏好"
- **设置弹窗**：
  - "登录后默认进入"：单选（月历/周历/日程表/日时间轴/看板总览）
  - "一周开始于"：单选（周一/周日）
  - "时间显示格式"：单选（12小时制/24小时制）
  - 保存后立即生效

## 数据模型（参考）

### 任务（Task）

```
Task {
  id: string              // 唯一标识
  company: string         // 公司名称（必填）
  position: string        // 岗位名称（必填）
  status: Status          // 状态枚举
  priority: Priority      // 优先级（用户可自定义）
  tags: Tag[]             // 标签数组（多标签）
  startDate: DateTime     // 开始日期（如投递日期）
  endDate: DateTime?      // 结束日期（如截止日期、面试日期）
  reminder: Reminder?     // 提醒设置
  attachments: Attachment[] // 附件列表
  notes: Note[]           // 笔记列表（支持多条，按时间排序）
  createdAt: DateTime     // 创建时间
  updatedAt: DateTime     // 更新时间
}

Status = 'preparing' | 'applied' | 'assessment' | 'interviewing' | 'waiting' | 'offered' | 'rejected'
Priority = { id, name, color, level } // 用户可自定义
Tag = { id, name, color }
Attachment = { id, name, type, url, size }
Note = { id, content, type, createdAt } // type: 'text' | 'interview_review' | 'follow_up'
Reminder = { type: 'email'|'push', time: DateTime }
```

### 快速备忘（ScratchNote）

```
ScratchNote {

  content: string        // 纯文本/富文本
  createdAt: DateTime
  status: 'draft' | 'converted'  // 草稿/已迁移为任务
  taskId: string?        // 迁移后的任务ID

```

## 备注

### 视图切换的动画设计

月历→周历→日时间轴的切换应给用户一种"镜头拉近"的空间感：
- **月历→周历**：当前月份视图向上淡出，周历从下方淡入+轻微放大（scale 0.98→1），营造从高空俯瞰到近景的过渡
- **周历→日**：类似"镜头继续拉近"，日历格子横向展开，显示更多时间细节
- **日→时间轴**：纵向展开为连续的时间流

### 空状态设计

所有视图和列表在数据为空时显示精心设计的空状态：
- **图标**：80px 插画（根据场景不同：空日历📅、空搜索🔍、空速记📝、空分析📊）
- **标题**：18px/600，如"还没有任何求职任务"
- **描述**：14px/Text Secondary，如"点击右上角 + 按钮开始记录你的第一个求职申请吧"
- **操作按钮**：Primary Blue 边框按钮，引导用户创建第一个内容

### 响应式考虑（基础）

本设计以桌面端为核心场景（大学生在电脑前管理求职），但基础响应式支持如下：
- **< 768px**：侧边栏自动折叠为图标模式，日历月历视图减少每行显示内容，详情抽屉变为全屏覆盖
- **768px - 1200px**：侧边栏可手动折叠，保持完整功能
- **> 1200px**：完整布局，侧边栏常驻

### 与竞品的差异化设计

| 功能 | 传统Excel | Teal/Huntr | 本产品设计 |
|------|----------|-----------|-----------|
| 时间视图 | ❌ 无 | ⚠️ 基础日历 | ✅ 四种专业视图 |
| 附件管理 | ❌ 外链 | ⚠️ 基础 | ✅ 完整附件系统+版本 |
| 面试复盘 | ❌ 无 | ⚠️ 基础笔记 | ✅ 结构化笔记+标签分类 |
| 快捷备忘 | ❌ 无 | ❌ 无 | ✅ 速记→草稿→任务迁移流 |
| 数据分析 | ❌ 手动统计 | ⚠️ 基础漏斗 | ✅ 多维度洞察+转化率 |
| 提醒通知 | ❌ 无 | ✅ 基础 | ✅ 智能截止日期提醒 |
| 标签系统 | ⚠️ 手动分类 | ✅ 预设标签 | ✅ 完全自定义标签+优先级 |

### 设计原则总结

1. **时间即秩序**：所有功能围绕时间线展开，日历是信息的组织原则而非附属功能
2. **减少认知负荷**：不展示不必要的信息，采用渐进式披露（hover/点击后才显示次要操作）
3. **为求职场景优化**：状态、颜色、字段命名都使用求职者的语言（"笔试中"而非"In Progress"）
4. **即时反馈**：拖拽、创建、更新操作都有明确的视觉反馈，让用户对系统状态有信心
5. **从记录到洞察**：不仅是记录工具，更通过数据分析帮助用户优化求职策略

## 月历视图（Month View）

### 概述

### 布局

- **网格**：6行 × 7列（周日-周六），每个格子等宽，高度自适应（最小 100px）
- **星期标题行**：7列顶部，文字为 Caption 12px/Text Tertiary/500，居中，高度 36px，底部边框 1px `#E8EBF0`
- **日期数字**：每个格子左上角，12px/600，当天日期用 Primary Blue 圆形背景（20px直径）+ 白色文字
- **格子样式**：
  - 默认：背景 `#FFFFFF`，右边框 1px `#E8EBF0`，下边框 1px `#E8EBF0`
  - 今天格子：顶部 2px Primary Blue 边框装饰
  - 非本月日期：背景 `#FAFBFC`，文字 Text Tertiary

### 布局

- **顶部日期行**：7列，每列宽度 equal（14.28%），高度 60px
  - 左边距：48px（留给时间轴刻度）
  - 每列内容：星期几（Caption/Text Secondary，如"周一"）+ 日期号（H2 18px/600，当天的日期号用 Primary Blue 圆形背景）
  - 底部边框 1px `#E8EBF0`
- **时间网格**：下方区域，24小时纵向时间轴
  - 左侧时间刻度：48px 宽，每2小时一个刻度（00:00, 02:00...），Caption/Text Tertiary，右对齐
  - 主体区域：7列等宽，每列48个30分钟时间段（每个15px高），背景 `#FFFFFF`
  - 当前时间线：横向红色线（1px `#EF4444`），带左侧红色圆点，实时更新位置
  - 网格线：横向每2小时一条线（1px `#F1F3F5`），每30分钟一条虚线（1px dashed `#F1F3F5`）

- 任务卡片根据其开始时间和持续时间定位在对应列的对应时间段上
- **卡片宽度**：单列内单个任务占满列宽（减去 4px margin）；同一时间段多个任务并排均分宽度
- **卡片高度**：根据持续时间计算（30分钟 = 15px）
- **卡片内容**：因空间更小，仅显示状态圆点+公司名（单行截断）+ 时间范围
- **全天任务**：固定在日期行下方，高度 24px，背景对应状态色20%，占满整列宽度

### 拖拽交互

- 任务卡片可在同列不同时间段拖拽（调整时间），也可拖拽到其他列（调整日期）
- 拖拽时显示半透明幽灵卡片在原位置，新位置实时预览
- 拖拽跨越天边界时，释放后日期和时间同时更新

## 看板总览视图（Board View）

### 概述

以状态为列的横向看板视图，每个求职任务是一张可拖拽的卡片，按申请进度从左到右排列。这是除时间视图外的核心管理模式，让用户一眼看清求职管道的整体健康状况。

### 布局

- **看板容器**：横向可滚动，固定列数 = 状态数量（7列）
- **每列**：
  - **宽度**：280px，不可调整
  - **列头**：高度 48px，padding 12px 16px

    - 右侧：+ 按钮（添加该状态下的新任务）

  - **列内容**：纵向滚动区域，padding 8px
    - 任务卡片垂直堆叠，间距 8px

- **顶部**：状态色顶部条（3px）+ 优先级竖条（左侧2px，中/高优先级显示）
- **公司名**：H3 15px/600/Text Primary，单行截断
- **岗位名**：14px/Text Secondary，单行截断
- **中间区**：
  - 标签列表（最多显示2个，超出"+N"）
  - 日期信息（如"投递：3月1日"或"截止：3月15日"），Caption/Text Tertiary

  - 左侧：附件图标+数量、笔记图标+数量
  - 右侧：头像占位（可分配给联系人）
- **Hover**：阴影增强为 shadow-md，显示快速操作按钮（编辑、删除、归档）

  - Toast 提示："[公司名] 已移动到 [新状态]"
  - 如果移动到"已Offer"，显示庆祝效果（卡片边框闪烁 Success Green，撒花动画从卡片顶部飘出，持续 1.5s）

### 双模式编辑的设计决策

任务详情采用"查看/编辑"双模式而非始终可编辑，原因如下：
1. **减少误触**：用户在查看详情时不会意外修改数据
2. **信息层级清晰**：查看模式下只展示有价值的信息，不被输入框干扰
3. **符合用户心智**：先阅读，再决定是否编辑
4. **移动端友好**：避免了触摸时意外触发编辑

### 笔记类型的设计意图

预定义三种笔记类型而非纯文本，是为了引导用户做有价值的记录：
- **文本**：通用备忘（如"记得更新简历中的项目经历"）
- **面试复盘**：结构化面试后反思（如"HR问了职业规划，回答得不够好，下次要准备2分钟的版本"），这类笔记积累下来就是面试题库
- **跟进记录**：与HR/面试官的沟通记录（如"3月5日发邮件询问进度，对方回复说下周给结果"）

用户还可以创建自定义类型（如"笔试经验"、"内推信息"），自定义类型可选择颜色。

## 备注

### 搜索性能优化

- 搜索输入使用 debounce（200ms），避免频繁请求
- 最少输入2个字符才触发搜索（防止单字母匹配过多结果）
- 服务端搜索+本地缓存，已加载的数据在本地做二次筛选
- 结果分页：首次加载20条，滚动到底部时加载更多

### 搜索快捷键设计

- `Ctrl/Cmd + K`：打开搜索浮层
- `Ctrl/Cmd + Shift + F`：打开完整搜索页面
- `↑↓`：在搜索结果中上下移动（浮层和完整页面通用）
- `Enter`：打开选中的结果
- `Esc`：关闭搜索浮层/返回上一页
- `/`：焦点直接移到搜索输入框（当在日历视图时）

### 高级搜索语法

支持简单的搜索语法让重度用户更快找到内容：

| 语法 | 说明 | 示例 |
|------|------|------|

| `tag:互联网大厂` | 按标签筛选 | `tag:互联网大厂` |
| `priority:高` | 按优先级筛选 | `priority:高` |
| `before:2025-03-15` | 截止日期之前 | `before:2025-03-15` |
| `after:2025-03-01` | 截止日期之后 | `after:2025-03-01` |

输入这些语法时，搜索框下方实时显示解析后的筛选条件预览（如"状态：面试中 · 关键词：产品经理"），帮助用户确认搜索意图。

# 快速备忘与草稿箱

## 概述

快速备忘是一个独立的轻量级记录空间，用于临时记录面试中灵光一闪的想法、HR电话中的关键信息、刷题时的知识点等碎片化内容。这些记录不需要立即组织成结构化任务，先存入草稿箱，后续可以在空闲时整理迁移到正式任务中。这个设计解决了"怕忘记先记下来"与"现在没时间整理"之间的矛盾。

## 入口方式

1. **顶部导航栏"速记"按钮**：始终可见，闪电图标+"速记"文字，点击打开速记输入浮层
2. **快捷键**：`Ctrl/Cmd + Shift + S` 全局打开速记浮层
3. **侧边栏"草稿箱"**：点击打开草稿箱管理页面，显示所有历史速记
4. **任务详情中的"迁移到速记"**：将正式任务退回草稿箱

## 速记输入浮层

### 触发与出现

- **触发**：点击导航栏"速记"按钮 / 快捷键 `Ctrl/Cmd + Shift + S`

- **背景**：`#FFFFFF`，圆角 xl，阴影 `shadow-xl`
- **动画**：从右下角滑入（scale 0.9→1 + opacity 0→1 + translateY 20px→0，250ms cubic-bezier(0.16, 1, 0.3, 1)）
- **层级**：z-index 最高，覆盖在所有内容之上

### 速记→任务迁移流程

用户点击"迁移到日历"后的完整流程：

1. **点击迁移按钮**
2. **解析弹窗**（可选步骤，如果速记内容足够结构化）：
   - 系统尝试从速记文本中提取关键信息（公司名称、日期、关键词）
   - 弹窗显示提取结果："检测到可能的公司名：字节跳动，是否使用？"
   - 用户确认或手动修正
3. **新建任务弹窗**：
   - 笔记内容自动填入"笔记"字段
   - 如果提取到公司名/岗位名，相应字段预填充
   - 用户补充其他信息（状态、日期、优先级等）
   - 点击"保存"后，速记状态变为"已归档"，并关联新创建的任务ID
4. **成功反馈**：
   - 草稿箱中该速记卡片状态变为"已归档"（绿色）
   - 显示"已关联到任务：[公司名-岗位名]"链接，点击打开任务详情
   - Toast："速记已成功迁移到日历"

### 多条速记合并迁移

批量操作中选中多条速记点击"迁移"时：
- 弹窗询问"合并为一条任务还是分别创建？"
- 合并：多条速记的笔记内容拼接为一条任务的笔记，用分隔线"---"隔开
- 分别：每条速记依次弹出新建任务弹窗（简化流程，只填公司和岗位即可保存，其余后续编辑）

### 速记的智能提醒

- 系统分析速记内容，如果检测到时间相关词汇（如"下周三"、"明天"、"3月15日"），在卡片上显示"🕐 检测到时间"提示
- 用户点击该提示可快速将速记转换为带有该日期的任务
- 如果检测到的日期已过期，显示红色警告"该日期已过期"

### 草稿箱的整理提醒

- 当"待整理"速记数量 ≥ 10 时，侧边栏"草稿箱"项右上角显示红色数字徽章
- 每周一（或用户自定义频率）推送一条温和的提醒："你有 N 条速记待整理，花5分钟整理一下吧"
- 提醒以 Toast 形式出现，带"去整理"和"稍后再说"按钮

### 速记内容的富文本支持

速记支持轻量级富文本格式（通过 Markdown 语法或快捷工具栏）：
- **粗体**：`Ctrl+B`
- **斜体**：`Ctrl+I`
- **链接**：自动识别 URL 变为可点击链接
- **@提及**：@ 后接公司名称可快速关联已有任务
- **#标签**：# 后接标签名可快速添加标签

富文本在速记卡片中渲染为格式化文本，迁移到任务时保持格式。

## 求职漏斗图

### 概述

以桑基图或垂直漏斗图展示求职各环节的人数流转，让用户一眼看清哪个环节是最大瓶颈。

### 布局位置

左侧卡片，占据约 55% 宽度，height 360px，圆角 lg，背景 `#FFFFFF`，阴影 shadow-sm，padding 20px。

### 图表设计

**漏斗图（垂直堆叠）**：
- 从上到下 7 个层级，对应 7 种状态
- 每个层级一个横向色条，宽度 = 该状态任务数 / 最大状态任务数 × 100%
- 色条使用对应状态色（准备中灰/已投递蓝/笔试中橙/面试中青/等结果紫/已Offer绿/已拒绝红）
- 色条右侧显示：状态名（14px/600）+ 数量（14px/Text Secondary）+ 转化率（Caption/Text Tertiary，如"28% 进入下一环节"）
- 色条之间有向下箭头连接（↓，24px/Text Tertiary）
- 最宽的一条（通常是"已投递"或"准备中"）为 100% 宽度，其余按比例缩放

**交互**：
- Hover 某层级：该层级色条宽度轻微放大（scaleX 1.02），显示 tooltip 包含详细数据（数量、转化率、平均停留天数）
- 点击某层级：跳转到看板总览的对应状态列

### 空状态

- 无数据时显示："还没有足够的数据生成漏斗图" + "开始记录求职任务后，这里会展示你的求职管道分析"

## 备注

### 数据计算逻辑

**漏斗转化率计算**：
- 准备中 → 已投递：在准备中停留后变为已投递的比例
- 已投递 → 笔试中：已投递后收到笔试的比例
- 笔试中 → 面试中：笔试通过后进入面试的比例
- 面试中 → 等结果：面试完成后等待结果的比例
- 等结果 → 已Offer：最终收到Offer的比例
- 已拒绝 = 任意状态变为已拒绝的比例（分流计算）

**周次定义**：
- 以用户设置的起始日期为基准（默认为首次创建任务的日期）
- 用户可在设置中修改"求职季起始日期"
- 自然周（周一到周日）

### 智能建议（扩展）

基于数据分析，系统可以在页面底部提供智能建议卡片：

- 如果面试通过率 < 10%："你的面试转化率偏低，建议优化简历或尝试内推渠道"
- 如果"等结果"状态堆积 > 5 个："有多个申请在等待反馈，建议主动发邮件跟进"
- 如果某标签通过率明显高于其他："互联网大厂类申请的通过率最高，可以适当增加此类投递"
- 如果本周投递量为0："本周还没有新的投递，保持节奏很重要"

建议卡片样式：
- 背景：Info Cyan 5%（信息类）或 Warning Orange 5%（提醒类）
- 左侧：对应颜色的竖条（3px）
- 内容：建议文字（14px/Text Primary）+ "了解更多"链接
- 可关闭（×按钮），关闭后24小时内不再显示同类建议

### 数据导出

页面右上角提供"导出报告"按钮：
- 点击后弹出选项：导出为 PDF / 导出为图片 / 导出为 CSV 数据
- PDF：生成格式化的求职报告，适合打印或发送给职业顾问
- 图片：生成所有图表的合成图片，适合分享到社交媒体（去敏处理后）
- CSV：原始数据，适合进一步分析

### 与其他页面的联动

洞察分析不是孤立页面，而是与整个产品形成闭环：
- 漏斗图 → 点击跳转到看板总览
- 折线图 → 点击跳转到周历视图
- 环形图/柱状图 → 点击跳转到搜索筛选
- 即将截止 → 点击打开任务详情
- 每个联动都在新标签页/视图打开，保留洞察页面的状态

### 空数据策略

当用户刚注册、任务数量 < 5 时：
- 显示"示例数据"模式：用模拟数据填充图表，让用户理解洞察页面的价值
- 图表上方半透明遮罩 + "这是示例数据，添加5个以上任务后查看你的真实分析"文字
- 示例数据用较浅的颜色（50%透明度），与真实数据区分
