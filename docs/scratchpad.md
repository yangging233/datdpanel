# 快速备忘与草稿箱

## 概述

快速备忘是一个独立的轻量级记录空间，用于临时记录面试中灵光一闪的想法、HR电话中的关键信息、刷题时的知识点等碎片化内容。这些记录不需要立即组织成结构化任务，先存入草稿箱，后续可以在空闲时整理迁移到正式任务中。这个设计解决了"怕忘记先记下来"与"现在没时间整理"之间的矛盾。

## 入口方式

1. **顶部导航栏"速记"按钮**：始终可见，闪电图标+"速记"文字，点击打开速记输入浮层
2. **快捷键**：`Ctrl/Cmd + Shift + S` 全局打开速记浮层
3. **侧边栏"草稿箱"**：点击打开草稿箱管理页面，显示所有历史速记
4. **任务详情中的"迁移到速记"**：将正式任务退回草稿箱

## 速记输入浮层

### 触发与出现

- **触发**：点击导航栏"速记"按钮 / 快捷键 `Ctrl/Cmd + Shift + S`
- **位置**：页面右下角固定位置（距底部 24px，距右侧 24px），宽度 380px
- **背景**：`#FFFFFF`，圆角 xl，阴影 `shadow-xl`
- **动画**：从右下角滑入（scale 0.9→1 + opacity 0→1 + translateY 20px→0，250ms cubic-bezier(0.16, 1, 0.3, 1)）
- **层级**：z-index 最高，覆盖在所有内容之上

### 浮层结构

```
┌──────────────────────────────────────────┐
│ ⚡ 快速备忘                      [—] [×] │  ← 头部 (drag handle + 最小化/关闭)
├──────────────────────────────────────────┤
│                                          │
│  [文本域 - 自动增高，placeholder:        │
│   "记下任何想法...面试要点、HR联系       │
│    方式、笔试题目、待办事项" ]           │
│                                          │
│                                          │
├──────────────────────────────────────────┤
│ 草稿箱(3)              [保存] [保存并创建]│  ← 底部栏
└──────────────────────────────────────────┘
```

### 头部区域

- **高度**：40px，padding 8px 12px，背景 `#F8F9FB`，圆角顶部 xl
- **左侧**：闪电图标 ⚡ + "快速备忘"（14px/600/Text Secondary）
- **右侧**：
  - 最小化按钮（—）：点击后浮层折叠为右下角小圆球（48px直径，Primary Blue背景+闪电白色图标），点击圆球恢复
  - 关闭按钮（×）：点击后如果文本域有内容，提示"是否保存到草稿箱？"
- **拖拽**：头部区域可拖拽，改变浮层位置（桌面端），位置记忆到 localStorage

### 文本域

- **样式**：
  - 无边框，背景透明
  - padding 12px 16px
  - 最小高度 120px，最大高度 300px（超出滚动）
  - 14px/400/Text Primary，行高 1.6
  - placeholder："记下任何想法...面试要点、HR联系方式、笔试题目、待办事项"
- **自动增高**：根据内容自动增高，直到最大高度
- **实时保存**：输入停止 2 秒后自动保存到草稿箱（debounce），右上角显示"已保存"微文字（Caption/Success Green，2秒后淡出）

### 底部栏

- **高度**：48px，padding 8px 12px，背景 `#F8F9FB`，顶部边框 1px `#E8EBF0`
- **左侧**："草稿箱(N)" 文字按钮（Caption/Primary Blue/500，N为未处理草稿数量），点击打开草稿箱页面
- **右侧**：
  - "保存"按钮（Primary Blue填充，小尺寸）：保存到草稿箱并清空文本域，可继续输入下一条
  - "保存并创建任务"按钮（Primary Blue边框，小尺寸）：保存后弹出新建任务弹窗，预填笔记内容

### 最小化态

- 浮层收缩为右下角圆形按钮（48px × 48px，圆角 full，背景 Primary Blue，阴影 shadow-md）
- 图标：白色闪电 ⚡
- 右上角小红点：有未处理草稿时显示（8px直径，Danger Red）
- **动画**：浮层 scale 1→0 + opacity 1→0（200ms），圆按钮 scale 0→1 弹出（200ms，弹性缓动）
- 点击圆按钮：反向动画恢复浮层

## 草稿箱管理页面

### 概述

草稿箱是速记内容的集中管理页面，显示所有历史速记（包括已处理和未处理），支持编辑、删除、批量迁移到正式任务。

### 页面结构

│ 侧边栏 │ 草稿箱                                                      │
│        │  ┌───────────────────────────────────────────────────────┐  │
│        │  │ 📝 草稿箱                          [批量操作] [+新建] │  │
│        │  │      "N 条速记，M 条待整理"                            │  │
│        │  ├───────────────────────────────────────────────────────┤  │
│        │  │                                                       │  │
│        │  │  ┌─────────────────────────────────────────────────┐  │  │

### 页面头部

- **标题**："📝 草稿箱"（H1 24px/600）
- **统计信息**："N 条速记 · M 条待整理"（14px/Text Secondary），待整理数量 M 用 Warning Orange 高亮
- **操作按钮**：
  - "批量操作"按钮（灰色边框）：点击后每条速记前出现复选框，顶部出现批量操作栏
  - "+ 新建速记"按钮（Primary Blue填充）：打开速记输入浮层

### 速记卡片

每条速记一个卡片，纵向排列，间距 12px。

- **尺寸**：width 100%，padding 16px，圆角 lg，背景 `#FFFFFF`，阴影 shadow-sm
- **顶部行**：
  - 复选框（批量操作模式下显示）
  - 创建时间（Caption/Text Tertiary，如"今天 15:30"、"昨天 10:15"、"3月5日"）
  - 右侧状态徽章："待整理"（Warning Orange）或"已归档"（Success Green）
- **内容区**：
  - 速记正文（14px/Text Primary，最多显示4行，超出"展开"按钮）
  - 展开后显示全文 + "收起"按钮
- **操作行**（margin-top 12px，hover 或总是显示）：
  - "编辑"按钮（小尺寸文字按钮，铅笔图标）
  - "迁移到日历"按钮（小尺寸文字按钮，日历图标，Primary Blue）— 点击后打开新建任务弹窗，笔记内容预填入"笔记"字段
  - "删除"按钮（小尺寸文字按钮，垃圾桶图标，Danger Red）
- **Hover**：阴影增强为 shadow-md

### 状态说明

- **待整理**：刚保存的速记，尚未迁移为正式任务或手动归档
- **已归档**：用户手动标记为"已处理"的速记，或已成功迁移为任务的速记（显示关联的任务链接）
- 状态切换：在编辑弹窗中可手动更改

### 批量操作

点击"批量操作"后：
- 每条速记前出现复选框
- 顶部出现批量操作栏（固定顶部，高度 48px，背景 Primary Light，阴影 shadow-sm）
- 批量操作栏内容：
  - 左侧："已选择 N 条" + 全选/取消全选
  - 右侧操作：
    - "批量迁移到日历"（Primary Blue边框）— 选择后打开任务创建流程，将多条速记合并为一个任务或分批创建
    - "批量归档"（灰色边框）— 标记为已处理
    - "批量删除"（Danger Red边框）— 弹出确认后删除

### 空状态

- 图标：📝 速记图标放大版（48px/Text Tertiary）
- 标题："草稿箱是空的"（H2 18px/600）
- 描述："使用速记功能快速记录想法，稍后再整理"（14px/Text Secondary）
- 操作："记一条速记"按钮（Primary Blue边框）— 打开速记浮层

## 备注

### 速记→任务迁移流程

用户点击"迁移到日历"后的完整流程：

1. **点击迁移按钮**
2. **解析弹窗**（可选步骤，如果速记内容足够结构化）：
   - 系统尝试从速记文本中提取关键信息（公司名称、日期、关键词）
   - 弹窗显示提取结果："检测到可能的公司名：字节跳动，是否使用？"
   - 用户确认或手动修正
3. **新建任务弹窗**：
   - 笔记内容自动填入"笔记"字段
   - 如果提取到公司名/岗位名，相应字段预填充
   - 用户补充其他信息（状态、日期、优先级等）
   - 点击"保存"后，速记状态变为"已归档"，并关联新创建的任务ID
4. **成功反馈**：
   - 草稿箱中该速记卡片状态变为"已归档"（绿色）
   - 显示"已关联到任务：[公司名-岗位名]"链接，点击打开任务详情
   - Toast："速记已成功迁移到日历"

### 多条速记合并迁移

批量操作中选中多条速记点击"迁移"时：
- 弹窗询问"合并为一条任务还是分别创建？"
- 合并：多条速记的笔记内容拼接为一条任务的笔记，用分隔线"---"隔开
- 分别：每条速记依次弹出新建任务弹窗（简化流程，只填公司和岗位即可保存，其余后续编辑）

### 速记的智能提醒

- 系统分析速记内容，如果检测到时间相关词汇（如"下周三"、"明天"、"3月15日"），在卡片上显示"🕐 检测到时间"提示
- 用户点击该提示可快速将速记转换为带有该日期的任务
- 如果检测到的日期已过期，显示红色警告"该日期已过期"

### 草稿箱的整理提醒

- 当"待整理"速记数量 ≥ 10 时，侧边栏"草稿箱"项右上角显示红色数字徽章
- 每周一（或用户自定义频率）推送一条温和的提醒："你有 N 条速记待整理，花5分钟整理一下吧"
- 提醒以 Toast 形式出现，带"去整理"和"稍后再说"按钮

### 速记内容的富文本支持

速记支持轻量级富文本格式（通过 Markdown 语法或快捷工具栏）：
- **粗体**：`Ctrl+B`
- **斜体**：`Ctrl+I`
- **链接**：自动识别 URL 变为可点击链接
- **@提及**：@ 后接公司名称可快速关联已有任务
- **#标签**：# 后接标签名可快速添加标签

富文本在速记卡片中渲染为格式化文本，迁移到任务时保持格式。
