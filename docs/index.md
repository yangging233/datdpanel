# 首页 — 时间视图看板

## 概述

首页是产品的核心页面，提供四种时间视图（月历、周历、日程表、日时间轴）的统一入口，以及看板总览视图。用户通过侧边栏或顶部导航切换视图，所有视图共享相同的导航框架，仅在内容区展示不同的任务排列方式。

## 页面结构

整体采用"侧边栏 + 主内容区"的左右布局，顶部有固定导航栏。

```
┌──────────────────────────────────────────────────────────────────────┐
│ [Logo] 求职看板      [月/周/日/时间轴]  [🔍] [⚡速记] [+ 新建] [👤] │  ← 顶部导航栏 (56px)
├────────┬─────────────────────────────────────────────────────────────┤
│ 侧边栏 │ 主内容区                                                    │
│        │                                                             │
│ ⏱ 月历 │  ┌───────────────────────────────────────────────────────┐  │
│ 📅 周历 │  │ 日历控制栏 [‹ ›  2025年3月  今天]  [筛选/排序]       │  │
│ 📋 日程 │  ├───────────────────────────────────────────────────────┤  │
│ 🕐 日轴 │  │                                                       │  │
│        │  │                  日历视图区域                          │  │
│ ─────  │  │                                                       │  │
│ 📊 看板│  │                  （四种视图切换）                       │  │
│ 📈 洞察│  │                                                       │  │
│ 📝 草稿│  │                                                       │  │
│ ─────  │  └───────────────────────────────────────────────────────┘  │
│ 🏷 标签│                                                             │
│        │                                                             │
└────────┴─────────────────────────────────────────────────────────────┘
```

### 顶部导航栏

已在 [design.md](design.md) 的"全局组件-顶部导航栏"中定义。首页的核心交互元素是视图切换标签组，位于导航栏中部。

**视图切换标签组细节**：
- 容器：圆角 full 的胶囊容器，背景 `#F1F3F5`，padding 2px
- 每个标签：padding 6px 16px，14px/500，圆角 full
- 选中态：背景 `#3B5BFF`，文字 `#FFFFFF`，过渡 200ms
- 未选中态：背景透明，文字 `#5A6270`，hover 文字 `#1A1D23`
- 切换时：当前选中标签背景色淡出，新标签背景色淡入

### 侧边栏

已在 [design.md](design.md) 的"全局组件-侧边栏"中定义。

**首页相关交互**：
- 点击"月历/周历/日程表/日时间轴"：切换主内容区视图，URL 同步更新（如 `/calendar/month`、`/calendar/week`）
- 点击"看板总览"：切换到看板视图（`/board`）
- 点击标签：当前视图追加按该标签筛选，标签高亮显示，视图标题变为"2025年3月 · 互联网大厂"，右上角出现"清除筛选"按钮

### 主内容区

内容区顶部有一个控制栏，下方是视图区域。

**日历控制栏**（高度 48px，padding 0 24px，背景透明）：
- **左侧**：日期导航
  - 左右箭头按钮（24px 图标按钮，hover 背景 Surface Hover）
  - 当前日期范围显示（如"2025年3月"、"3月10日-16日"、"3月12日"），H2 18px/600
  - "今天"按钮（点击跳回当前日期）
- **右侧**：视图控制
  - 筛选按钮（漏斗图标+文字"筛选"，点击展开筛选下拉）
  - 排序按钮（排序图标，下拉选项：按日期/按优先级/按状态/按公司名）
  - 密度切换（仅日程表视图显示：紧凑/舒适）

## 月历视图（Month View）

### 概述

传统的月网格视图，每个格子代表一天，任务以彩色卡片形式展示在对应日期上。这是最宏观的视图，适合了解全月求职节奏。

### 布局

- **网格**：6行 × 7列（周日-周六），每个格子等宽，高度自适应（最小 100px）
- **星期标题行**：7列顶部，文字为 Caption 12px/Text Tertiary/500，居中，高度 36px，底部边框 1px `#E8EBF0`
- **日期数字**：每个格子左上角，12px/600，当天日期用 Primary Blue 圆形背景（20px直径）+ 白色文字
- **格子样式**：
  - 默认：背景 `#FFFFFF`，右边框 1px `#E8EBF0`，下边框 1px `#E8EBF0`
  - 今天格子：顶部 2px Primary Blue 边框装饰
  - 非本月日期：背景 `#FAFBFC`，文字 Text Tertiary
  - Hover：背景 Surface Hover

### 任务卡片展示

- 每个格子内从日期数字下方开始堆叠任务卡片
- **最多显示**：前3个任务卡片，超出显示"+N 更多"按钮（12px/Primary Blue/500，点击切换到日时间轴查看该天）
- **卡片样式**：使用全局"任务卡片"组件样式
- **全天任务**：卡片左侧有"全天"标识，不显示具体时间
- **截止时间提醒**：距离截止日期 ≤3天的任务卡片右上角显示橙色小点（6px）；≤1天显示红色小点+轻微脉动动画（pulse，2s infinite）

### 空状态

- 当月没有任何任务时，网格中央显示空状态插画+"这个月还没有求职任务"
- "+ 新建第一个任务"按钮，点击打开新建弹窗，日期自动填入今天

### 新建任务交互

- **点击空白格子**：在该格子的日期位置出现一个浮动快速新建卡片（200px宽），包含公司名和岗位名两个输入框，输入后按 Enter 或点击外部保存
- **右键空白格子**：上下文菜单 → 新建任务 / 新建全天任务 / 粘贴速记

### 拖拽交互

- 任务卡片可在格子间拖拽
- 拖拽经过的格子显示虚线边框（2px dashed Primary Blue）
- 释放后任务日期更新，Toast 提示"已移动到 3月15日"

## 周历视图（Week View）

### 概述

展示连续7天的横向时间网格，每天一列，可看到具体时间段的任务分布。适合安排面试、协调笔试与其他事务的时间冲突。

### 布局

- **顶部日期行**：7列，每列宽度 equal（14.28%），高度 60px
  - 左边距：48px（留给时间轴刻度）
  - 每列内容：星期几（Caption/Text Secondary，如"周一"）+ 日期号（H2 18px/600，当天的日期号用 Primary Blue 圆形背景）
  - 底部边框 1px `#E8EBF0`
- **时间网格**：下方区域，24小时纵向时间轴
  - 左侧时间刻度：48px 宽，每2小时一个刻度（00:00, 02:00...），Caption/Text Tertiary，右对齐
  - 主体区域：7列等宽，每列48个30分钟时间段（每个15px高），背景 `#FFFFFF`
  - 当前时间线：横向红色线（1px `#EF4444`），带左侧红色圆点，实时更新位置
  - 网格线：横向每2小时一条线（1px `#F1F3F5`），每30分钟一条虚线（1px dashed `#F1F3F5`）

### 任务展示

- 任务卡片根据其开始时间和持续时间定位在对应列的对应时间段上
- **卡片宽度**：单列内单个任务占满列宽（减去 4px margin）；同一时间段多个任务并排均分宽度
- **卡片高度**：根据持续时间计算（30分钟 = 15px）
- **卡片内容**：因空间更小，仅显示状态圆点+公司名（单行截断）+ 时间范围
- **全天任务**：固定在日期行下方，高度 24px，背景对应状态色20%，占满整列宽度

### 拖拽交互

- 任务卡片可在同列不同时间段拖拽（调整时间），也可拖拽到其他列（调整日期）
- 拖拽时显示半透明幽灵卡片在原位置，新位置实时预览
- 拖拽跨越天边界时，释放后日期和时间同时更新

## 日程表视图（Agenda/List View）

### 概述

按时间倒序排列的列表视图，以日期为分组展示所有求职任务。适合快速浏览最近的求职活动和即将到来的安排。

### 布局

- **列表容器**：纵向滚动，每条日期为一个分组
- **日期分组头**：高度 40px，背景 `#F8F9FB`，左边距 16px
  - 日期文字（如"3月12日 周三"），H3 15px/600
  - 当天任务数量徽章（Caption，背景 `#E8EBF0`，Text Secondary）
  - 底部分隔线 1px `#E8EBF0`
- **任务行**：高度 64px，padding 12px 16px，左右结构
  - **左侧时间区**（80px宽）：开始时间（14px/Text Primary）+ 结束时间（12px/Text Tertiary）
  - **中部指示条**：4px宽，对应状态色，圆角 full
  - **右侧内容区**：
    - 第一行：状态徽章 + 公司名称（H3 15px/600）+ 岗位名称（14px/Text Secondary）
    - 第二行：标签圆点+名称（Caption）+ 优先级指示（🔴🟡）+ 附件数量图标+数字
  - **右侧操作区**（hover显示）：编辑铅笔图标 + 更多菜单（⋮）
  - 底部分隔线 1px `#F1F3F5`
  - Hover：背景 Surface Hover

### 截止时间高亮

- 今日及之后的日期分组：正常显示
- 已过去的日期：背景 `#FAFBFC`，日期文字 Text Tertiary
- 有截止日期提醒的日期分组头：右侧显示"⚠️ 3天后截止"等提示文字（Warning Orange）

### 排序与分组

- 默认按日期倒序（最新的在前）
- 排序切换选项：按日期升序/降序、按优先级、按状态、按公司名A-Z
- 当日任务按时间顺序排列，无时间的任务放在最前面

## 日时间轴视图（Day Timeline View）

### 概述

单日的纵向时间流视图，以30分钟为粒度展示一天24小时，所有任务按实际时间定位。适合面试当天的时间管理和准备节奏把控。

### 布局

- **日期标题**：顶部固定区域，H1 24px，如"3月12日 周三"，当天显示"今天"标签（Primary Blue徽章）
- **时间轴**：纵向滚动，24小时，每30分钟一个刻度段
  - 左侧时间标签：48px宽，每整点显示一次（如"09:00"），Caption/Text Tertiary
  - 主体区域：任务卡片定位区
  - 当前时间线：横向红线（同周历视图），实时位置
  - 背景：每小时交替使用 `#FFFFFF` 和 `#FAFBFC`（便于区分时段）
  - 刻度线：每30分钟一条（1px `#F1F3F5`）

### 任务展示

- 任务卡片按实际开始时间定位，高度按持续时间计算
- **卡片宽度**：单个任务占满可用宽度（减去 16px margin）；重叠任务并排均分
- **卡片内容**：最丰富的展示，包含：
  - 状态色顶部条（3px）
  - 状态徽章 + 优先级图标
  - 公司名称（H3 15px/600）
  - 岗位名称（14px/Text Secondary）
  - 时间范围（Caption/Text Tertiary）
  - 标签列表（彩色小圆点+名称，可横向滚动）
  - 笔记预览（最多2行，12px/Text Secondary，截断）
  - 附件图标+数量
- **全天任务**：固定在时间轴顶部，单独的横条区域

### 面试准备模式

- 日时间轴有一个特殊模式"面试日"（当天有面试任务时自动建议开启）
- 开启后：
  - 面试任务卡片扩展，显示"准备清单"子区域（checkbox列表：查公司资料/准备自我介绍/准备问题/测试设备/提前10分钟上线）
  - 面试前30分钟卡片边框变为脉动的 Warning Orange（提醒即将开始）
  - 面试进行中卡片边框变为 Success Green 实线

## 看板总览视图（Board View）

### 概述

以状态为列的横向看板视图，每个求职任务是一张可拖拽的卡片，按申请进度从左到右排列。这是除时间视图外的核心管理模式，让用户一眼看清求职管道的整体健康状况。

### 布局

- **看板容器**：横向可滚动，固定列数 = 状态数量（7列）
- **每列**：
  - **宽度**：280px，不可调整
  - **列头**：高度 48px，padding 12px 16px
    - 左侧：状态色圆点（8px）+ 状态名称（14px/600）+ 任务数量徽章（Caption）
    - 右侧：+ 按钮（添加该状态下的新任务）
    - 底部边框 2px 状态色
  - **列内容**：纵向滚动区域，padding 8px
    - 任务卡片垂直堆叠，间距 8px
- **列顺序**（从左到右）：准备中 → 已投递 → 笔试中 → 面试中 → 等结果 → 已Offer → 已拒绝

### 看板卡片

比日历视图更丰富的卡片设计：

- **尺寸**：width 100%（264px），min-height 100px，padding 12px
- **背景**：`#FFFFFF`，圆角 md，阴影 shadow-sm
- **顶部**：状态色顶部条（3px）+ 优先级竖条（左侧2px，中/高优先级显示）
- **公司名**：H3 15px/600/Text Primary，单行截断
- **岗位名**：14px/Text Secondary，单行截断
- **中间区**：
  - 标签列表（最多显示2个，超出"+N"）
  - 日期信息（如"投递：3月1日"或"截止：3月15日"），Caption/Text Tertiary
- **底部区**：
  - 左侧：附件图标+数量、笔记图标+数量
  - 右侧：头像占位（可分配给联系人）
- **Hover**：阴影增强为 shadow-md，显示快速操作按钮（编辑、删除、归档）
- **拖拽**：opacity 0.7，scale 1.03，阴影 shadow-lg

### 拖拽变更状态

- 卡片可拖拽到任意其他列
- **拖拽过程**：经过的列显示虚线边框（2px dashed 该列状态色），背景变为该状态色的5%
- **释放后**：
  - 卡片状态更新为列对应状态
  - 卡片在新列中以动画形式就位（从释放位置飞入目标位置，300ms）
  - Toast 提示："[公司名] 已移动到 [新状态]"
  - 如果移动到"已Offer"，显示庆祝效果（卡片边框闪烁 Success Green，撒花动画从卡片顶部飘出，持续 1.5s）

### WIP 限制提示

- 每列可设置最大卡片数（默认不限，用户可自定义）
- 当列中卡片数超过限制时：
  - 列头数量徽章变为 Warning Orange
  - 底部出现提示条："此列已有 N 个任务，建议控制在 M 个以内"
  - 新拖拽到该列的卡片有橙色边框警告

### 空列处理

- 没有任务的列显示空占位：
  - 列内容区居中显示虚线框（2px dashed `#E8EBF0`，圆角 md）
  - 文字"拖拽任务到此处"（Caption/Text Tertiary）
  - 或点击"+"按钮新建该状态的任务

## 动画

| 动画 | 触发 | 细节 |
|------|------|------|
| 视图切换过渡 | 切换月/周/日/时间轴 | 当前视图 opacity 1→0（150ms），新视图 opacity 0→1 + translateY 8px→0（200ms） |
| 月历→周历 | 点击"周历" | 镜头拉近感：月历淡出同时scale 1→0.95，周历淡入scale 0.98→1 |
| 卡片hover | 鼠标移入卡片 | translateY -1px + 阴影增强（200ms ease-out） |
| 拖拽反馈 | 开始拖拽 | opacity 0.7 + scale 1.02 + 增强阴影 |
| 拖拽放置 | 经过可放置区 | 区域显示虚线边框 + 浅色状态背景 |
| 状态变更庆祝 | 卡片移到"已Offer" | 卡片边框闪烁绿色 + 顶部飘出彩色纸屑（1.5s） |
| 截止日期提醒 | 截止时间≤1天 | 卡片右上角红点脉动动画（2s infinite，ease-in-out） |
| 当前时间线 | 每分钟 | 红线位置平滑过渡（60s线性） |
| 快速新建出现 | 点击日历格子 | 浮动卡片 scale 0.9→1 + opacity 0→1（200ms） |
| 空状态出现 | 数据为空时 | 插画+文字 opacity 0→1 + translateY 20px→0（300ms） |
| 看板列拖拽 | 卡片飞入新列 | 从释放位置到目标位置，300ms cubic-bezier(0.32, 0.72, 0, 1) |

## 响应式

- **< 768px**：
  - 侧边栏完全隐藏，通过汉堡菜单呼出（底部浮层）
  - 月历视图：格子最小高度 60px，任务卡片仅显示状态条+公司名首字母
  - 周历视图：隐藏（移动端不支持，自动切换到日程表）
  - 日程表视图：全宽列表，时间区缩小到 48px
  - 日时间轴视图：时间标签隐藏，任务卡片全宽
  - 看板视图：单列纵向滚动（每列一个section），不可横向拖拽

- **768px - 1024px**：
  - 侧边栏折叠为图标模式（72px）
  - 看板列宽缩小到 240px

- **> 1024px**：
  - 完整布局

- 背景：Success Green 10%，圆角 lg，padding 12px 16px
- 内容："你今天有 N 场面试" + "进入面试准备模式"按钮（Primary Blue边框）
- 点击按钮后：日时间轴切换为面试准备模式（见上文"日时间轴视图-面试准备模式"）

### 截止日期提醒策略

不同时间段的视觉提醒强度应呈梯度递增：
- > 7天：无特殊标记
- 3-7天：任务卡片右上角橙色小点（静态）
- 1-3天：橙色小点 + 卡片边框变为 Warning Orange（1px）
- < 1天：红色小点脉动 + 卡片边框 Danger Red + 在时间轴上显示倒计时（"还剩6小时"）

### 看板视图与时间视图的联动

- 在"准备中"列新建任务时，默认 startDate 为空（收藏意向）
- 将卡片从"准备中"拖拽到"已投递"时，自动将 startDate 设为今天
- 将卡片拖拽到"已Offer"或"已拒绝"时，自动将 endDate 设为今天（标记流程结束）
- 反之，在时间视图中将任务的截止日期改为过去，看板中该任务列显示"已过期"提示

### 快捷操作效率设计

为重度用户设计的效率操作：
- 双击日历空白处 = 快速新建（仅填公司名和岗位名）
- 右键任务卡片 = 上下文菜单：编辑/变更状态/复制/删除/迁移到速记
- 键盘导航：在日程表视图中，↑↓键在任务间移动，Enter打开详情，Delete删除（需确认）
- 批量操作：在日程表中按住 Shift 或 Ctrl/Cmd 多选任务，顶部出现批量操作栏（变更状态/添加标签/删除）
