# 求职申请管理看板

一个基于 `Vite + React + Tailwind CSS` 的前端原型项目，面向求职场景，提供任务看板、时间视图、全局搜索、草稿箱、笔记库和数据洞察等能力。

当前版本以本地前端交互为主，数据默认保存在浏览器 `localStorage`，适合用于产品演示、界面迭代、交互验证和后续接入后端。

## 项目特点

- 面向求职流程设计，围绕“准备中、已投递、简历筛选、笔试中、面试中、已 Offer、已拒绝”等状态组织任务
- 首页同时支持 `看板 / 月历 / 周历 / 日程表 / 日时间轴` 多视图切换
- 支持任务详情抽屉、新建任务、拖拽改状态/日期等核心交互
- 提供全局搜索与多条件筛选
- 提供草稿箱，可将临时记录迁移为正式任务
- 提供笔记库，集中查看和管理任务下的笔记
- 提供洞察分析页，展示状态分布、投递趋势、转化漏斗等统计信息
- 设计文档较完整，适合继续做产品化开发

## 页面概览

### 1. 首页

首页是核心工作区，默认进入看板视图，并可切换到：

- 看板视图
- 月历视图
- 周历视图
- 日程表视图
- 日时间轴视图

支持的主要操作：

- 查看任务状态分布
- 打开任务详情抽屉
- 新建任务
- 按状态/优先级筛选
- 通过拖拽调整任务状态或日期

### 2. 洞察分析

用于统计和复盘求职进度，当前包含：

- 核心指标卡片
- 状态分布图
- 近 12 周趋势图
- 转化漏斗
- 优先级分布
- 最近更新列表
- JSON 数据导出

### 3. 全局搜索

支持按以下维度检索任务：

- 公司
- 岗位
- 描述
- 标签
- 状态
- 优先级
- 截止日期范围

### 4. 草稿箱

用于快速记录临时想法、岗位线索或待整理信息，后续可一键迁移为正式任务。

### 5. 笔记库

聚合所有任务内的笔记，支持：

- 按类型筛选
- 按内容/公司/岗位搜索
- 编辑笔记
- 删除笔记
- 跳转到关联任务

## 项目结构

```text
F:\datapanel
├─ docs/                  产品设计与页面说明
├─ src/
│  ├─ components/         业务组件、布局组件、基础 UI 组件
│  ├─ hooks/              自定义 Hook
│  ├─ lib/                常量、工具函数、本地数据读写
│  ├─ pages/              页面级组件
│  ├─ App.jsx             路由入口
│  └─ main.jsx            应用启动入口
├─ index.html
├─ package.json
├─ tailwind.config.js
└─ vite.config.js
```

其中几个关键目录：

- `src/pages`：当前页面包括首页、洞察分析、搜索、草稿箱、笔记库
- `src/components/views`：首页中的多种视图组件
- `src/lib/data.js`：基于 `localStorage` 的任务、草稿、笔记数据读写
- `docs/`：产品设计稿和各页面详细说明

## 运行要求

- Node.js `18`
- npm 或 yarn

项目根目录中存在 `.nvmrc`，当前版本为 `18`。

## 本地启动

### 使用 nvm

```bash
nvm install 18
nvm use 18
```

### 安装依赖

```bash
npm install
```

### 启动开发环境

```bash
npm run dev
```

默认使用 Vite 本地开发服务器。

## 可用脚本

```bash
npm run dev
npm run build
npm run build:dev
npm run lint
npm run preview
```

说明：

- `dev`：启动开发服务器
- `build`：构建生产包
- `build:dev`：以 development 模式构建
- `lint`：执行 ESLint 检查
- `preview`：预览构建结果

## 数据说明

当前项目未接入后端，主要数据保存在浏览器本地：

- 任务：`jobtracker_tasks`
- 草稿：`jobtracker_scratchpad`

首次进入时会自动写入一组默认示例任务。

这意味着：

- 刷新页面后数据仍会保留
- 更换浏览器或清空本地存储后数据会丢失
- 现阶段更适合原型演示而非正式生产使用

## 当前实现边界

目前仓库更偏向高保真前端原型，已具备较完整 UI 和部分交互，但仍有这些边界：

- 无真实后端接口
- 无登录与权限系统
- 无数据库持久化
- 无附件上传后端存储
- 无自动化测试
- 部分设计文档中的高级交互尚未完全落地

## 后续优化建议

- 接入真实后端与数据库，替换 `localStorage`
- 为任务、笔记、附件建立标准 API
- 增加用户体系、数据同步和权限控制
- 补充单元测试和基础集成测试
- 统一 README、设计文档与实际实现之间的状态说明
